# key_switcher

Небольшой набор скриптов для работы с выделенным или скопированным текстом через буфер обмена.

Поддерживает:
- исправление текста, набранного в неправильной раскладке EN/RU
- инвертирование регистра
- автоматическое переключение системной раскладки KDE после исправления текста

Скрипты лежат по путям:
- `/path/key_switcher/switch_layout.py`
- `/path/key_switcher/run.sh`

## Как это работает

`run.sh`:
- берет текст из буфера обмена через `wl-paste`
- передает его в `switch_layout.py`
- кладет результат обратно в буфер через `wl-copy`
- при обычной смене раскладки дополнительно переключает системную раскладку KDE через `qdbus`

`switch_layout.py`:
- определяет направление конвертации по большинству букв в каждом токене
- корректно обрабатывает спорные символы вроде `.` `,` `?` в зависимости от контекста слова
- умеет отдельно инвертировать регистр через аргумент `--invert-case`

## Зависимости

Нужно, чтобы в системе были доступны:
- `python3`
- `wl-copy`
- `wl-paste`
- `qdbus`

Проверка:

```bash
which python3 wl-copy wl-paste qdbus
```

## Основные команды

### 1. Исправить раскладку текста

Команда:

```bash
/path/key_switcher/run.sh
```

Что делает:
- исправляет текст в буфере обмена
- переключает системную раскладку KDE на нужную (`us` или `ru`)

Пример:

```text
ntcnjdjt ,kjrjt 'cct
```

Результат:

```text
тестовое блокое эссе
```

### 2. Инвертировать регистр

Команда:

```bash
/path/key_switcher/run.sh --invert-case
```

Что делает:
- меняет `ПрИвЕт` на `пРиВеТ`
- меняет `HeLLo` на `hEllO`
- системную раскладку не переключает

## Вызов напрямую через Python

Если нужно использовать только сам Python-скрипт:

### Смена раскладки

```bash
echo "ntcnjdjt ,lkjrjt 'cct" | python3 /path/key_switcher/switch_layout.py
```

### Инвертирование регистра

```bash
echo "ПрИвЕт HeLLo" | python3 /path/key_switcher/switch_layout.py --invert-case
```

### Определение целевой раскладки

```bash
echo "ntcnjdjt" | python3 /path/key_switcher/switch_layout.py --print-target-layout
```

Возможные результаты:
- `ru`
- `us`
- пустая строка, если направление не удалось определить

## Горячие клавиши в KDE

Открыть:
- `System Settings` -> `Shortcuts` -> `Custom Shortcuts`

### Горячая клавиша для исправления раскладки

Команда:

```bash
/path/key_switcher/run.sh
```

### Горячая клавиша для инвертирования регистра

Команда:

```bash
/path/key_switcher/run.sh --invert-case
```

Логично назначить две разные комбинации, например:
- исправление раскладки: `Meta+Alt+L`
- инвертирование регистра: `Meta+Alt+U`

## Как пользоваться

1. Выделить текст и скопировать его в буфер обмена.
2. Нажать нужную горячую клавишу.
3. Вставить текст обратно.

## Лог

Для отладки wrapper-скрипт пишет вывод в файл:

```bash
/tmp/keyswitcher.log
```

Посмотреть:

```bash
cat /tmp/keyswitcher.log
```

Очистить:

```bash
rm -f /tmp/keyswitcher.log
```

## Примечания

- Скрипт ориентирован на Wayland.
- Автоматическое переключение раскладки сделано под KDE.
- Список раскладок берется из файла:

```bash
/home/valeriy/.config/kxkbrc
```

- Сейчас ожидается порядок раскладок:

```text
us,ru
```

Если порядок раскладок в KDE изменится, `run.sh` все равно попытается найти нужную раскладку по имени из `LayoutList`.
